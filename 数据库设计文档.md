# 校园二手交易平台 - 数据库设计文档

| 项目 | 说明 |

## 3. 关系模型

### 3.1 实体关系说明

| 关系 | 类型 | 说明 |
|:---|:---|:---|
| user → goods (发布) | 1:N | 用户作为卖家发布商品 |
| user ↔ goods (收藏) | M:N | 用户收藏商品，落地生成 favorite 中间表（含收藏时间） |
| user → order (购买) | 1:N | 用户作为买家创建订单（buyer_id） |
| user → order (售出) | 1:N | 用户作为卖家接收订单（seller_id） |
| user → review (发表) | 1:N | 用户作为买家发表评价（reviewer_id） |
| user → review (被评) | 1:N | 用户作为卖家收到评价（seller_id） |
| user → admin_log (记录操作) | 1:N | 管理员产生操作日志（admin_id） |
| category → goods (包含商品) | 1:N | 一个分类下可包含多个商品 |
| goods → goods_image (包含图片) | 1:N | 一个商品可有多张展示图片 |
| goods → order (订单商品) | 1:N | 一个商品可能产生多个历史订单 |
| order → review (评价所属) | 1:1 | 一个订单有且仅有一条评价，评价依附于订单 |

## 4. 表结构设计（数据字典）

### 4.1 用户信息表 (user)

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|:---|:---|:---|:---|:---|
| user_id | INT UNSIGNED | PK, AUTO_INCREMENT | - | 用户编号 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | - | 用户名 |
| password | VARCHAR(255) | NOT NULL | - | 密码(bcrypt加密) |
| real_name | VARCHAR(50) | NOT NULL | - | 真实姓名 |
| student_id | VARCHAR(20) | NOT NULL, UNIQUE | - | 学号 |
| phone | VARCHAR(15) | NOT NULL | - | 手机号 |
| email | VARCHAR(100) | - | NULL | 邮箱 |
| wechat | VARCHAR(50) | - | NULL | 微信号 |
| region | VARCHAR(50) | NOT NULL | - | 所在区域(北区/南区) |
| avatar | VARCHAR(255) | - | NULL | 头像路径 |
| role | ENUM('user','admin') | - | 'user' | 用户角色 |
| status | ENUM('active','banned') | - | 'active' | 账号状态 |
| created_at | DATETIME | - | CURRENT_TIMESTAMP | 注册时间 |
| updated_at | DATETIME | - | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- PRIMARY KEY (`user_id`)
- UNIQUE KEY `uk_username` (`username`)
- UNIQUE KEY `uk_student_id` (`student_id`)
- KEY `idx_region` (`region`)

### 4.2 商品分类表 (category)

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|:---|:---|:---|:---|:---|
| category_id | INT UNSIGNED | PK, AUTO_INCREMENT | - | 分类编号 |
| category_name | VARCHAR(50) | NOT NULL, UNIQUE | - | 分类名称 |
| description | VARCHAR(255) | - | NULL | 分类描述 |
| sort_order | INT | - | 0 | 排序号 |
| created_at | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |

**索引：**
- PRIMARY KEY (`category_id`)
- UNIQUE KEY `uk_category_name` (`category_name`)

### 4.3 商品信息表 (goods)

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|:---|:---|:---|:---|:---|
| goods_id | INT UNSIGNED | PK, AUTO_INCREMENT | - | 商品编号 |
| seller_id | INT UNSIGNED | NOT NULL, FK | - | 卖家编号 |
| category_id | INT UNSIGNED | NOT NULL, FK | - | 分类编号 |
| title | VARCHAR(100) | NOT NULL | - | 商品标题 |
| description | TEXT | - | NULL | 商品描述 |
| price | DECIMAL(10,2) | NOT NULL | - | 价格 |
| original_price | DECIMAL(10,2) | - | NULL | 原价 |
| region | VARCHAR(50) | NOT NULL | - | 所在区域 |
| condition_level | ENUM('new','like_new','used','old') | - | 'used' | 新旧程度 |
| status | ENUM('on_sale','sold','off_shelf') | - | 'on_sale' | 商品状态 |
| view_count | INT UNSIGNED | - | 0 | 浏览量 |
| contact_phone | VARCHAR(15) | - | NULL | 联系电话 |
| contact_wechat | VARCHAR(50) | - | NULL | 微信号 |
| created_at | DATETIME | - | CURRENT_TIMESTAMP | 发布时间 |
| updated_at | DATETIME | - | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

## 5. 完整性约束设计

### 5.1 实体完整性
- 所有表均设置主键，且主键字段不允许为空
- 用户表：username、student_id 设置唯一约束
- 分类表：category_name 设置唯一约束
- 订单表：order_no 设置唯一约束
- 收藏表：user_id + goods_id 设置联合唯一约束
- 评价表：order_id 设置唯一约束（一个订单只能有一条评价）

### 5.2 参照完整性
- 商品表外键：`fk_goods_seller` → 删除用户时级联删除其发布的商品；`fk_goods_category` → 删除分类时限制，防止误删有关联商品的分类
- 订单表外键：删除用户或商品时级联删除相关订单
- 评价表外键：删除订单时级联删除相关评价

### 5.3 用户定义完整性
- 价格字段：DECIMAL(10,2)，确保金额精度
- 评分字段：TINYINT UNSIGNED，范围 1-5
- 状态字段：ENUM 类型，限制取值范围
- 手机号字段：VARCHAR(15)，预留国际号码格式
- 时间字段：DATETIME，自动记录创建和更新时间

## 7. 数据库实施

### 7.1 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS campus_secondhand
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;

USE campus_secondhand;
```

### 7.2 执行建表脚本

```bash
mysql -u root -p < schema.sql
```

如需导入演示用初始数据，可在建表完成后执行：

```bash
mysql -u root -p campus_secondhand < init_data.sql
```

### 7.3 验证表结构

```sql

DESC user;
DESC goods;
-- 查看其他表...
```
